Projet Shell-HTML

Présentation du projet

Le but de ce projet est la génération d'une galerie d'images à partir d'un répertoire d'images sources.
Le programme possède six options:
  • --source: choix du répertoire source contenant les images à présenter dans la galerie,
  • --dest: choix du répertoire destination, existant ou non,
  • --index: choix du nom du fichier où sera généré le code de la galerie,
  • --force: forcer la création des vignettes,
  • --help affichage de l'aide,
  • --verb: mode verbeux détaillant le déroulant du programme.
Le code du projet est composé de deux scripts shell et de deux feuilles de style CSS. Le script galerie_shell.sh prépare la génération de la galerie. Il permet la récupération des options et arguments passés en ligne de commande. Ensuite, le script utilities.sh génère la galerie d'image.

Rôle des fonctions de utilities.sh

  • html_head
    • Génère l'entête HTML d'un fichier donné en paramètre avec le titre de la page.
    La feuille de style CSS est concaténée dans l'entête. Elle peut être soit celle de la galerie "galerie_style.css", soit celle des images en taille réelle "image_style.css".
  • html_title
    • Génère un titre sur le fichier HTML donné en paramètre.
  • html_tail
    • Génère le pied de page sur le fichier HTML donné en paramètre.
  • generate_img_fragment
    • Génère le fragment de code HTML pour afficher la vignette dans la galerie.
  • generate_navi
    • Génère la navigation entre les pages avec les images en taille réelle : boutons "Précèdent", "Suivant" et "Retour à la galerie"
  • generate_img_html
    • Correspond à la fonction "galerie_main" mais pour les images en taille réelle. Génère la page HTML associée à l'image en taille réelle.
  • generate_galerie
    • Génère les vignettes associées aux images sources. Affichage de chaque vignette dans la galerie et création de la page associée à l'image réelle.
  • galerie main
    • Génère la galerie d'images.

Tests

Les tests se lancent avec la commande ./tests/nom_du_test depuis le dossier du projet. Pour tous les tests sauf force.sh, il faut vérifier manuellement que la galerie générée correspond à celle attendue. Dans le cas du test force.sh sur l'option force, la vérification est effectuée par le test et le résultat est affiché dans le terminal.
